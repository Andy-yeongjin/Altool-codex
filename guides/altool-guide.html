🥚 4단계 — 개발 시작
🥚

/al oneshot — 명령어 하나로 완주

만들고 싶은 기능을 한국어로 설명하면 6단계가 자동으로 돌아갑니다

Claude Code 대화창에 붙여넣기 (기능 설명만 바꾸세요)
/al oneshot 여기에 만들고 싶은 기능을 설명하세요
✍️
예시:
/al oneshot 포모도로 타이머. 25분 집중, 5분 휴식 사이클. 시작/정지 버튼, 오늘 완료 횟수 표시.
/al oneshot 방명록. 이름과 메시지를 남기면 최신순으로 보이고 새로고침해도 유지.
/al oneshot 가계부. 수입/지출 입력, 카테고리별 분류, 월별 차트.
AI가 자동으로 실행하는 6단계
0/6 자산 감지 + 📚 교훈 회고 (헌법·디자인·PRD·lesson)
1/6 개발 계획 (plan)
2/6 상세 설계 — 아키텍처 3안 비교 (design)
3/6 코드 구현 + 빌드 검증 (run)
4/6 갭 분석 — 독립 검증 에이전트 (analyze)
5/6 자동 개선 — 갭이 있으면 (fix)
6/6 브라우저 기능·디자인 검증 + 자체 수정
🥚 단계 시작→ 🐣 단계 완료→ 🐥 사이클 완주
⏱️
시간이 좀 걸립니다 (10~30분). 멈추는 경우는 둘뿐입니다: ① 6단계에서 개발 서버가 꺼져 있으면 → start.bat 실행 후 "계속" 입력, ② 자동 개선 2회 후에도 갭이 남으면 → 선택지를 줍니다.
✅ 5단계 — 결과 확인
✓

start.bat으로 직접 보기 + 문서로 검증 확인

프로젝트 폴더의 start.bat을 더블클릭 → 브라우저에서 localhost:3000

docs/01-plan/ 계획서→ 02-design/ 설계서→ 03-analyze/ 갭 분석→ 04-report/ 완료 보고서
📋
문서가 진행 상태의 원천입니다. 단계가 끝날 때마다 체크박스(- [x])와 Status가 자동 갱신됩니다 — 계획서만 열어도 뭐가 끝났고 뭐가 남았는지 보입니다. 검증 안 된 항목은 체크되지 않습니다.
💬
수정은 그냥 대화로: "로그인 버튼 색 바꿔줘", "글자가 너무 작아" — 헌법·디자인 토큰 검증이 자동으로 따라붙습니다.
🔁 단계별 수동 명령 — 세밀하게 제어하고 싶을 때
명령어역할산출물
/al plan 기능명계획 — 요구사항·성공 기준·리스크docs/01-plan/…/기능명.plan.md
/al design 기능명설계 — 아키텍처 3안·API 계약·UI 체크리스트docs/02-design/…/기능명.design.md
/al run 기능명구현 — Depth-First + 빌드 검증소스 코드
/al analyze 기능명검증 — 독립 에이전트 갭 분석 (4축 Match Rate)docs/03-analyze/기능명.analyze.md
/al fix 기능명개선 — 갭이 있으면 수정→재검증 (최대 5회)개선 코드 + 기능명.fix.md
/al report 기능명보고 — 전체 사이클 요약 + 회고docs/04-report/기능명.report.md
/al status현황 — 단계·Match Rate·최근 이력🪺 출력만
🐣
헌법·디자인 토큰 강제는 엔진에 내장되어 있어 긴 지시문을 붙일 필요가 없습니다 — /al plan 로그인 한 줄이면 됩니다. oneshot은 위 단계들을 자동으로 이어서 실행하는 것뿐입니다.
📚 lesson — 같은 실수를 반복하지 않는 시스템
📚

오류는 자동으로 기록되고, 다음 작업에서 자동으로 회고됩니다

모든 프로젝트가 공유하는 글로벌 교훈 노트 (~/.altool/lesson.md)

lesson 생명주기
오류 해결 →  자동 기록 (증상/원인/해결/예방 규칙)
plan·run 시작 →  자동 회고 (📚 관련 교훈 반영)
analyze →  검증 렌즈 (과거 결함 패턴 검사)
수동으로 조회·기록할 때
/al lesson

⏸️ 중간에 멈췄어요▼

Claude가 안내 메시지를 줍니다 — 대부분 질문에 답하면 진행됩니다. 대표적인 멈춤: 서버 필요(→ start.bat 실행 후 "계속"), PRD/디자인 파일이 여러 개(→ 하나 선택), 개선 2회 후 갭 잔존(→ A/B 선택지).

📊 Match Rate가 뭔가요?▼

설계 문서와 실제 코드의 일치율입니다. 독립 검증 에이전트(코드를 짠 AI와 다른 별도 AI)가 구조·기능·API 계약·런타임 4축으로 측정합니다. 90% 이상이 통과선이지만, 갭이 1건이라도 발견되면 점수와 무관하게 수정합니다 — 90%는 통과선이지 수정 면제선이 아닙니다.

🏛️ 헌법이 뭔가요? 바꿀 수 있나요?▼

프로젝트 루트의 constitution.md — TypeScript Strict, 보안, 접근성(터치 44px·대비 4.5:1), 디자인 토큰 강제 등 19개 조항입니다. 모든 단계가 자동으로 준수하며, 파일을 직접 수정하면 커스터마이징됩니다.

📦 altool/ 폴더만 다른 프로젝트에 복사해도 되나요?▼

됩니다. altool/ + .claude/commands/al.md 두 가지만 있으면 동작합니다. 헌법·디자인 파일이 없으면 해당 규칙만 비활성화되고(1회 안내) 나머지는 정상 작동합니다.
