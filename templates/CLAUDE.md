# 프로젝트 개발 환경 (Altool Powered)

> **Altool** — 헌법·디자인 시스템·개발 사이클을 하나로 합친 통합 개발 시스템.
> 플러그인 설치 없이 `/al` 명령어 하나로 계획→설계→구현→검증→보고를 완주합니다.

---

## ⚡ 세션 시작 — 반드시 먼저 실행

**세션이 시작되면 아래 파일을 즉시 읽어라. 어떤 작업을 하기 전에 먼저 읽어야 한다.**

1. `constitution.md` — 개발 헌법 전문. 모든 조항을 이 프로젝트의 모든 작업에 적용한다.
2. `altool/CLAUDE.md` — Altool 엔진 공통 규칙.

읽은 후:

- **기존 프로젝트 재개**: `/al setup`
- **단계 파악이 필요할 때**: `/al guide`
- **새 기능 바로 시작**: `/al oneshot [기능 설명]`

---

## 🗺️ 개발 워크플로우

```
[0] /al design_source   ← (선택) .pen/Stitch → designs/design.md + design-tokens.css
[1] /al plan     ─┐
[2] /al design    │
[3] /al run        ├─ /al oneshot [기능 설명] 하나로 자동 완주
[4] /al analyze   │   (독립 검증 에이전트 + Match Rate 4축)
[5] /al fix   │   (갭 발견 시, 자동)
[6] 브라우저 검증  ─┘
[7] /al report   ← 완료 보고서
```

헌법·디자인 토큰 강제는 **엔진에 내장**되어 있다 — 명령어에 지시문을 붙일 필요 없음.

---

## ⛔ 절대 규칙 — 모든 결과물은 헌법 준수 재검증 필수

**코드 수정 요청이 오면 응답하기 전에 반드시 `constitution.md`, `designs/design.md`, `designs/design-tokens.css`를 Read 도구로 직접 열어 확인한 후 작업하라.** UI 수정이라면 해당 토큰(`var(--토큰명)`)을 사용하고, 토큰에 없는 값은 토큰을 추가한 후 사용한다. 검증 없이 "완료"라고 말하는 것은 허용되지 않는다.

### 핵심 4계명

1. **디자인 토큰만 사용**: `color: #2563EB` ❌ → `color: var(--color-primary-600)` ✅
2. **서버 자동 실행 금지**: 개발 완료 후 `start.bat` 실행을 안내하고 대기
3. **빌드 검증 = 완료 조건**: `npm run build` 성공 없이 "완료" 보고 금지
4. **오류 해결 = lesson 기록**: 원인 규명이 필요했던 오류를 해결하면 즉시 `~/.altool/lesson.md`에 4줄 형식(증상/원인/해결/예방 규칙)으로 기록 — `/al` 명령 실행 중이든 일상 수정 중이든 동일 적용

## 🛠️ Altool 명령어

| 명령어 | 설명 | 언제? |
|--------|------|-------|
| `/al setup` | 초기 설치 + 세션 재개 | 처음 또는 돌아올 때 |
| `/al oneshot [기능설명]` | 6단계 자동 파이프라인 (PRD·.pen 자동 감지) | 새 기능을 한방에 |
| `/al guide` | 현재 단계 감지 + 다음 명령 안내 | 뭘 할지 모를 때 |
| `/al design_source` | .pen/Stitch → 디자인 시스템 생성 | 디자인 확정할 때 |
| `/al lesson [내용]` | 교훈 기록·조회 (글로벌 누적, plan·run 시 자동 회고) | 같은 실수를 반복하기 싫을 때 |
| `/al plan·design·run·analyze·fix·report {기능명}` | 개발 사이클 단계별 수동 실행 | 세밀하게 제어할 때 |
| `/al status` | 기능별 진행 현황 + 최근 이력 | 상태 확인 |

> `.pen` 파일은 `designs/` 폴더에, Stitch ZIP은 풀어서 `designs/stitch/`에 넣으세요.
> PRD는 `prd/`에, 참고 문서는 `prd/refs/`에 넣으면 자동 감지됩니다.

## 🏛️ 개발 헌법 핵심 요약

> 전체 원문: `constitution.md` (19개 조항)

| 조항 | 원칙 |
|------|------|
| 제0조 | 모든 표준 문서는 한글로 작성 |
| 제0조의2 | Asia/Seoul 기준, YYYY-MM-DD 형식 |
| 제1조 | 명세 우선 (SDD) — 계획 문서가 Single Source of Truth |
| 제2조 | 보편 언어 (DDD) — 기획·설계·코드·DB 동일 용어 |
| 제3조 | 도메인 무결성 — 비즈니스 로직 격리 |
| 제5조 | Vercel React Best Practices 준수 |
| 제6조 | TypeScript Strict, 보안, 세션 검증 |
| 제8조 | 검증 의무 — 테스트 미통과 코드는 미완성 |
| 제9조 | 명세-코드 동기화 — 코드 수정 시 문서 먼저 |
| 제10조 | 프로덕션 빌드 성공 = 완료 |
| 제15조 | 메인 화면 우선, 비인증 탐색 보장 |
| 제16조 | 디자인 토큰 필수, 하드코딩 금지 |
| 제17조 | .pen은 구조 원본, 수치는 토큰으로 교정 |
| 제18조 | 로컬 SQLite → 배포 전 DB 결정(ADR) |
| 제19조 | 정부 디자인 헌법 — 대비 4.5:1, 터치 44px, 키보드 접근성 |
