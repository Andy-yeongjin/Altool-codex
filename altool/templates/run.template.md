## 2. Depth-First 구현 규칙 ⚠️ CRITICAL

> **깊게 구현하라, 넓게 펼치지 마라.**
> 안티패턴: 20개 파일 스켈레톤 → 각 30% 완성 (구조는 90%인데 기능은 빈껍데기)
> 올바른 패턴: **배치당 3~5개 파일을 100% 구현** → 검증 → 다음 배치

배치 사이클:
```
① 하위 레이어(타입·도메인) 구현 → ② 즉시 검증(단위/curl)
③ 상위 레이어(훅·컴포넌트·페이지) 구현 → ④ 즉시 검증(L2/렌더)
⑤ 전부 green → 다음 배치
```

파일별 완성 기준 (다음 파일로 넘어가기 전 확인):
- `// TODO`·플레이스홀더 없음, `console.log` 스텁 핸들러 없음
- 하드코딩 목 배열 없음 (실데이터 또는 빈 상태 처리)
- Design §5.4의 해당 요소 전부 존재 + 실제 핸들러 연결
- 로딩·에러 상태 구현
